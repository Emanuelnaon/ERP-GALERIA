NANO POS - Hoja de Ruta (Galería Esquina, Corrientes)
📌 Estado Actual: FASE 4 (Cierre de MVP y Pruebas de Estrés)
Fecha Límite MVP: 31 de Marzo
Objetivo Principal: Reemplazar el cuaderno y el Excel. El sistema debe permitir cobrar de forma ágil, descontar stock exacto y registrar las ventas del turno sin fallas.

✅ Fase 1 a 3: Arquitectura, Core y Seguridad (COMPLETADAS)
[x] Configuración de Supabase, Autenticación y Despliegue en Vercel.

[x] POS Básico: Buscador, Carrito, y Cobro rápido (F10).

[x] Arquitectura "Mostaza": Inicio de sesión ultrarrápido con DNI.

[x] Bloqueo por Hardware: Bautismo de terminales físicas limitadas por local.

[x] Integración de lector de código de barras USB (Auto-Focus y Modo Supermercado).

[x] Pestaña "Mis Ventas" con historial en vivo para seguridad del cajero.

🟡 Fase 4: Integración Crítica y Pulido (EN PROGRESO - Hasta el 22 de Marzo)
(Foco exclusivo en blindar el flujo de caja. Cero desarrollo de módulos administrativos nuevos).

[ ] Lógica estricta de Stock: Definir y probar la reacción del sistema al intentar vender en negativo (ej. si el stock es 0, ¿bloquea la venta o avisa y permite continuar?).

[ ] Manejo de errores del Lector USB: Configurar la respuesta exacta de la interfaz (alerta visual/sonora) si se escanea un código que no existe en la base de datos.

[ ] Flujo 100% Teclado: Asegurar que el cajero pueda completar una venta entera (escanear + F10 + Confirmar) sin necesidad de tocar el mouse.

🚀 Fase 5: Destrucción Controlada y Entrega (23 al 31 de Marzo)
[ ] Limpieza de Producción: Purgar base de datos (borrar ventas, cajas y gastos de prueba).

[ ] Carga Inicial: Alta manual (directo en Supabase) de los usuarios reales y contraseñas para el día de apertura.

[ ] Pruebas de Estrés (QA): Jugar a ser "el peor cajero". Escanear 20 productos por segundo, intentar borrar ítems mientras carga, apretar F10 repetidamente.

[ ] Despliegue Final: Entrega del sistema configurado en las PCs de la galería.

🔮 Fase 6: NANO POS v1.0 (ROADMAP POST-MVP - Objetivo: 30 de Junio)
(Módulos de valor agregado que se construirán con el cliente ya operando).

[ ] Módulo de Gestión de Personal (UI): Pantalla en el Dashboard para altas, bajas y reseteo de contraseñas de empleados sin usar Supabase.

[ ] Reportes Contables Exportables: Panel para filtrar ventas y descargar Excel.

[ ] Módulo de Ingreso de Mercadería: Interfaz visual para sumar stock de proveedores.
